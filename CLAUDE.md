# AgenticChunkingLibrary

ODC External Logic component for Level 5 Agentic Chunking workflows.
Provides three stateless actions consumed by the Agentic Chunking ODC application.

## Contribution rules

- **No AI attribution anywhere.** Do not add `Co-Authored-By`, `Generated by`, or any other AI tool credit to commit messages, code comments, or documentation. All commits must be authored solely by the human committer.
- **No AI mentions in code or docs.** Do not reference AI assistants, LLM tools, or code-generation tools in source files, test files, or any markdown in this repository.

## What this library does

- **PreChunkForExtraction** — splits source text into token-safe batches before the first AI Gateway call
- **ParsePropositions** — parses the extraction LLM's JSON response into a clean list of proposition strings; handles markdown fences, doubled quotes, and outer-quoted responses
- **NormaliseAgenticOutput** — parses the grouping LLM's JSON response into `AgenticChunk` structs

## What this library does NOT do

- Call the AI Gateway or any external service
- Store state between calls
- Validate business rules (min/max chunk size, document policies) — those belong in ODC
- Depend on ChunkingLibrary or SemanticChunkingLibrary

## ODC rules — non-negotiable

These constraints come from the OutSystems External Libraries SDK. Breaking any of them produces an upload error.

| Rule | Reason |
|------|--------|
| All model types must be `struct` | `OS-ELG-MODL-05015` if class is used |
| All model structs must have `[OSStructure]` | Required for ODC type mapping |
| Interface must have `[OSInterface]` | `OS-ELG-MODL-05001` if missing |
| All actions must have `[OSAction]` | `OS-ELG-MODL-05002` if missing |
| Implementing class must have a public parameterless constructor | ODC instantiates it directly |
| No static state | Calls are stateless; static fields would bleed between requests |
| No async/await on interface methods | ODC does not support async actions |
| Target framework: net8.0 | Required by ODC External Logic runtime |
| DLLs must be at the root of the zip | `OS-ELG-MODL-05027` if inside a subdirectory |

## Known ODC upload error codes

| Code | Cause |
|------|-------|
| `OS-ELG-MODL-05027` | DLL is inside a subdirectory in the zip. Use `zip -j` to strip paths. |
| `OS-ELG-MODL-05015` | A model type is a `class`. Change to `struct`. |
| `OS-ELG-MODL-05002` | Interface decoration is wrong. Check `[OSInterface]`, `[OSAction]`, `[OSParameter]`. |
| `OS-ELG-MODL-05001` | No `OSInterface` found. Verify the interface is public and decorated. |

## SDK namespace

```csharp
using OutSystems.ExternalLibraries.SDK;
```

## Token estimation

One token ≈ 4 characters throughout this library. `TokenEstimator.Estimate(text)` returns `text.Length / 4`. This is intentionally simple — the ODC application controls the actual prompt budget.

## Test coverage scope

Unit tests cover only C# behaviour:
- Batching logic and token envelope handling (`PreChunkForExtraction`)
- JSON parsing, fence stripping, doubled-quote normalisation, outer-quote stripping (`ParsePropositions`)
- JSON parsing, struct field mapping, failure modes, markdown fence stripping (`NormaliseAgenticOutput`)

**Out of scope for unit tests** (require live AI Gateway calls, run in the ODC demo app):
- Proposition extraction quality: pronoun resolution, compound sentence splitting (TC-001 to TC-010)
- Grouping quality: domain separation, vocabulary overlap (TC-016 to TC-020)
- Edge cases requiring LLM reasoning (TC-027 to TC-029)
- Full pipeline integration (TC-031 to TC-035)
